# Smart Volume (智能音量) 插件

这是一个为 Ulanzi Deck 编写的智能音量控制插件，支持多模式音量调节与可视化的屏幕悬浮窗反馈。

> **注意：本插件仅支持 Windows 操作系统。**

## 功能特性

1. **常规旋转**：调整 Windows 系统主音量。
2. **按下并旋转**：智能识别当前系统处于**前台激活状态**的应用，并仅针对该应用调整音量（非常适合在玩游戏或看视频时，独立调节当前窗口的音量而不影响系统其他软件的声音）。
3. **双击呼出进程列表**：
   - 双击旋钮会呼出一个包含所有当前活跃音频进程的列表界面（系统主音量固定在列表顶部）。
   - 在列表模式下：
     - **正常旋转**：在列表中上下滚动选中光标。
     - **按下旋转**：直接调整当前光标对应进程的音量。
     - **单击旋钮**：锁定当前选中的进程（选中项会变成金色高亮）。
     - **锁定后旋转**：此时无需一直按压旋钮，直接正常左右旋转即可调节被锁定进程的音量。再次单击解除锁定。
     - **自动退出**：如果在 3 秒内没有任何操作，列表会自动淡出并恢复到常规模式。
     - **双击退出**：双击旋钮，列表会淡出并恢复到常规模式。
4. **OSD 悬浮窗显示**：每次调节音量时，会在屏幕底部显示现代化的半透明音量进度条及进程列表。

## 安装与使用

### 安装

1. 前往本项目的 Releases 页面下载最新版本的插件压缩包。
2. 解压后，将文件夹（名称需确保为 `com.lsby.smart-volume.ulanziPlugin`）放置于 Ulanzi Deck 的插件目录中。默认路径为：
   `%AppData%\Ulanzi\UlanziDeck\Plugins\`
3. 完全退出并重新启动 Ulanzi Deck 客户端以加载该插件。

### 使用方法

1. 在 Ulanzi Deck 客户端界面中，选中设备上的任意一个**旋钮**按键。
2. 在右侧属性设置栏中，将“智能音量 (Smart Volume)”插件的动作分配给该旋钮。
3. 分配成功后即可生效，具体操作手势请直接参考上方的**功能特性**说明。

## 核心组件说明

目前该插件的所有音频控制底层已由 C# 的 Windows CoreAudio API 原生接管，彻底去除了对外部 exe 的命令调用，实现了零延迟和极度顺滑的体验：

- `app.js`: 插件主逻辑（Node.js），负责接收 Ulanzi WebSocket 事件，并通过命名管道 (Named Pipes) 与 OSD 进程极速通信。
- `src/app-main.cs`: OSD 程序的入口与命名管道服务端通信逻辑。
- `src/osd-window.cs`: 屏幕音量悬浮窗 (OSD) 以及进程列表模式的 WPF 界面布局、渲染与交互状态机管理。
- `src/audio-manager.cs`: 封装了系统底层的 CoreAudio API 接口，负责音频会话抓取与音量控制。

## 编译方法

如果您修改了 `src/` 目录下的 C# 源码文件，需要重新编译生成可执行程序才能生效。本插件自带了基于 Windows 内置 `.NET Framework` 的编译脚本，**无需安装 Visual Studio 或其他庞大的开发环境**。

### 方法 1：使用批处理 (推荐)

1. 进入 `scripts` 文件夹。
2. 双击运行 `build.bat`。
3. 脚本会自动调用系统的 `csc.exe` 编译器，并将编译成功的 `volume-osd.exe` 文件直接输出到 `bin` 文件夹下。

### 方法 2：使用 PowerShell

1. 打开 PowerShell 终端。
2. 定位到 `scripts` 目录。
3. 执行 `.\build.ps1` 命令。
4. 编译结果同样会输出至 `bin` 文件夹。

## 开发与调试

- 插件的日志开关及输出受 `config.json` 控制。将 `debug` 字段设置为 `true` 后，运行日志会输出到 `logs/plugin.log` 文件中。如遇功能异常，请开启调试模式并查看日志。
- OSD 渲染程序 `volume-osd.exe` 会在 `app.js` 启动时以子进程身份预热并拉起，并实现生命周期绑定（随 Node.js 一起启动与销毁）。
